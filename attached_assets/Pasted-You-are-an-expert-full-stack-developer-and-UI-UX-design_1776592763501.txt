FEATURE 1 — AUTO-AUTHENTICATION ONBOARDING
────────────────────────────────────────
Problem: New users currently have to manually copy credentials from the Firebase
console to authenticate. This creates friction and confusion.
Implement:
On first app launch or sign-up, automatically provision and authenticate the user
using Firebase Authentication behind the scenes (email/password or Google OAuth).
Store the Firebase config securely in environment variables (FIREBASE_API_KEY,
FIREBASE_AUTH_DOMAIN, FIREBASE_PROJECT_ID, etc.) so users never see or touch it.
On successful registration, immediately redirect the user to their personalized
dashboard without any intermediate configuration step.
If using Google Sign-In, use Firebase's signInWithPopup or signInWithRedirect —
the user just clicks "Continue with Google" and lands on their dashboard.
Show a branded loading/welcome screen during auth initialization instead of a
blank screen.
Handle edge cases: expired tokens (auto-refresh), first-time users (create Firestore
user document automatically), returning users (restore their session).
Tech: Firebase Auth SDK v9+ modular API. Use onAuthStateChanged listener to gate
dashboard rendering.

FEATURE 7 — NOTEBOOKLM-STYLE FEATURE IN STUDY HUB
────────────────────────────────────────
Add a new "Study Notebook" feature inside Study Hub, replicating the core
functionality of Google's NotebookLM. This is the most complex feature.
CORE CAPABILITIES TO IMPLEMENT:
A. SOURCE UPLOAD & MANAGEMENT
Users can create a "Notebook" for each subject or topic.
Each notebook accepts up to 10 sources per session:
· PDF upload (parse with PDF.js or server-side)
· Google Docs link (via Docs API or paste text)
· Plain text / copy-paste
· YouTube URL (extract transcript via YouTube Data API)
· Website URL (fetch and extract readable content)
Show source cards in a left sidebar with title, type icon, and page/word count.
Allow users to enable/disable individual sources (checkbox toggle).
Sources are stored per notebook in Firestore under the user's document.
B. AI-GENERATED NOTEBOOK GUIDE (CENTER PANEL)
When sources are added, auto-generate using the Gemini API:
· A "Notebook Guide" — a structured summary of all sources combined
· Key topics list (clickable, jumps to relevant source passage)
· Suggested questions the user might want to explore
· A timeline (if sources have temporal content)
Display this in a clean, readable center panel with section headers.
Add a "Regenerate" button to refresh the guide when sources change.
C. AUDIO OVERVIEW (Deep Dive Podcast)
Generate an "Audio Overview" — a realistic two-voice podcast-style conversation
summarizing the notebook content, using the Gemini TTS API or Web Speech API.
Show a player UI: play/pause button, waveform visualizer (animated bars),
playback speed control (0.75×, 1×, 1.25×, 1.5×, 2×), and progress bar.
Allow users to download the audio as an MP3.
Show a transcript below the player (auto-scroll as audio plays).
D. INLINE CHAT / Q&A (RIGHT PANEL)
A chat interface grounded on the uploaded sources only.
User types a question → AI answers using only the notebook's sources.
Each AI response includes inline citations like [Source 1, p.4] that are
clickable and highlight the relevant passage in the source.
Support follow-up questions (maintain conversation context).
"Copy response" and "Save to notes" buttons on each AI message.
Show "Answer not found in sources" gracefully if the question goes outside scope.
E. NOTEBOOK MANAGEMENT
Users can create multiple notebooks (one per subject recommended).
Notebook list on Study Hub home: shows title, source count, last edited date.
Rename, duplicate, and delete notebooks.
Share a notebook with classmates (generate a shareable link).
TECH STACK FOR THIS FEATURE:
Gemini API (gemini-1.5-pro or gemini-2.0-flash) for summarization and Q&A
Gemini TTS or Web Speech API for audio
Firebase Firestore for notebook/source storage
Firebase Storage for uploaded PDFs
PDF.js for client-side PDF text extraction
Streaming responses for the chat (use Gemini streaming API)
UI LAYOUT (three-panel, responsive):
Mobile: tabbed (Sources | Guide | Chat)
Tablet: two panels (Sources sidebar + Guide/Chat main)
Desktop: three panels side by side (Sources | Guide | Chat)
Header: Notebook title (editable inline) + source count + "Audio Overview" button
